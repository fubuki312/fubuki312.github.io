Java位移符的使用

运算符 运用
<< 左移 3<<N = 3*2的N次方
>> 右移 3>>N = 3*2的-N次方
>>> 无符号右移 3>>> N = 3⁄2
& 与运算 6 & 3 = 2
| 或运算 6 | 3 = 7
^ 异或运算 6 ^ 3 = 5
~ 反码 ~6 = -7

位运算，都是直接对二进制进行运算

首先我们了解什么是二进制

根据百度的二进制百度百科，一个二进制的大小为1byte（字节），1byte为8位，1位有4个0或1组成

所以可以得出，一个完整的二进制格式为

0000 0000 0000 0000 0000 0000 0000 0000

任何数据都是以二进制的方式存储到电脑上的，那么是如何将数据转换为2进制的呢？以数字66为例

就是将66一直除以2，直到商为0或1时为止，如果除数 余1或整除为0，那就把余记下来，就得到了我们的二进制码，再将二进制码从下往上排列，得到0100 0010。可以看到我在1前面补了一个0，为什么要补0呢，因为前面提到1byte为8位，每一位有4个0或1组成，但是66除下来是1000010，明显没有8个，所以我们就要在前面用0替补，一个完整的二进制码应如下

0000 0000 0000 0000 0000 0000 0100 0010

需记住，正数的二进制码，如数量不够，则使用0来凑

二进制具体的不再赘述，只是简单介绍下二进制，接下来我们介绍位移符是如何对二进制进行操作的

左位移和右位移

惯例上图，左位移右位移，就是将二进制码中的值向左或向右边移动，超出红线（1byte长度）的祛除掉，长度不够就凑0，位移多少位就补多少个0

记二进制太麻烦，我们直接用公式来算会更好

只需记住

左移：m << n = m*2^n

右移：m >> n = m*2^-n

接下来我们来看看无符号右移

无符号右移

首先注意，只有无符号右移，没有无符号左移

当无符号右移的值为正数的时候，运算方式与右移一致

例：8 >>> 2 = 2 = 8 >> 2

当无符号右移的值为负数的时候，运算方式就不一样了

例 ： -8 >> 2 = -2，而 -8 >>> 2 = 1073741822

为什么呢？我们来看图，简单的看下使用负数进行右位移和无符号右移的差别

当使用负数右位移的时候，最高位补的是1，为什么呢？其实也并不一定是补1，而是看移动后，在未进行部位的情况下，看最前面的数字是啥，如果是0则补0，是1则补1

而使用无符号右移呢，则前面的补位为正负号，如果是1则是负数，是0则是正数，这也是为什么它叫无符号右移，就是位移后进行补位的都是0，不会有负数存在。

与运算&和或运算|

与运算看图就能看出大概的一个逻辑，就是二进制码进行对比，如果同位有0，那么新的位置上就写0，如果俩个都是1，则写1

或运算则是和与运算恰恰相反的，是如果同位有1，就写1，俩个都是0则写0。俩者简单易懂。

异或运算^

看图，可以得出规律：同位都是0或都是1的时候，写0，其他情况都是1

反码~

反码比较简单，就是0和1互相转换，0变成1,1变成0
